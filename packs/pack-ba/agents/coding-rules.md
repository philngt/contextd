# pack-ba — Coding Rules

Quy tắc viết/chuẩn hóa tài liệu BA.

## Requirement Format

- Viết requirement theo cấu trúc: actor, trigger, business outcome.
- Ưu tiên bullet points ngắn cho assumptions, dependencies, out-of-scope.

## Acceptance Criteria Format

- Dùng format testable (Given/When/Then hoặc tương đương).
- Mỗi tiêu chí nên có owner xác nhận hoặc nguồn xác minh.
